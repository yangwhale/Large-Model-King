# 孕兽之道


![孕兽之道](../../assets/ch14b-pretraining.png)
*一切驯兽之法，都建立在一个前提之上——你得先有一头兽。兽从何来？从兽卵来。卵怎么孵？用海量灵食日夜灌注。这个过程叫预训练，是修仙界最烧钱、最基础、也最容易被忽视的一步。*

---

## 一

篇四讲的是怎么让神兽听话——PPO、DPO、GRPO，各种御兽之法层出不穷。

但在讲怎么驯兽之前，得先回答一个更根本的问题：**兽从哪来？**

修仙界的训练流水线分三步：

**第一步，预训练（孵化）。** 把一颗混沌未分的兽卵，用海量灵食日夜灌注，直到它破壳而出。这一步花掉了整个训练预算的 95% 以上。GPT-4 的预训练据估算花了超过一亿美元。DeepSeek V3 精打细算也花了 557 万美元。

**第二步，SFT（驯兽）。** 预训练出来的幼兽有知识但不听话。SFT 用几万条精心标注的示范数据教它"人类问你的时候应该怎么回答"。相当于教一头野兽基本的规矩。

**第三步，RLHF/GRPO（结灵契）。** 用强化学习让神兽学会理解人类的真正意图，不只是"格式正确"，而是"真正有用"。

后面两步，第十五章到第十九章会详细讲。本章只讲第一步——**孵化**。

因为没有孵化，后面一切都是空谈。

## 二

预训练的核心，简单到让人不敢相信。

**预测下一个智元。**

就这么一件事。

给你"今天天气"四个字，猜第五个字是什么。答案是"真"。再给你"今天天气真"，猜第六个是什么。答案是"好"。

从左到右，一个字一个字往后猜。猜对了，Loss（蒙昧值）降一点。猜错了，调整凝智元（参数），下次争取猜对。

几万亿次猜测之后，神兽学会了什么？

表面上看——它学会了"哪个字通常跟在哪个字后面"。这像是统计模式匹配，好像没什么了不起。

但奇妙的事情发生了。

为了准确预测下一个字，神兽**不得不**理解语法——因为语法决定了哪些词可以出现在当前位置。它**不得不**理解语义——因为"昨天我去了医院"后面大概率不会接"所以我很开心"。它**不得不**理解逻辑——因为"如果 A 那么 B"的推理链中，每一步都有确定的下一个字。它**不得不**理解世界知识——因为"中国的首都是"后面只能接"北京"。

**一个看似愚蠢的训练目标，迫使神兽学会了语言的一切。**

这就是为什么 Ilya Sutskever（须菩提祖师）曾经说过一句让人深思的话："如果你能完美地预测下一个 Token，你就理解了产生这些 Token 的世界。"

当然，"完美预测"是不可能的——因为语言本身有随机性。"今天天气真"后面可以接"好"也可以接"差"也可以接"热"。但越大的模型、用越多的数据训练，就越接近"完美预测"。而在逼近的过程中，神兽的灵智就会越来越高。

这就是 Scaling Law 的本质——不是因为大就好，而是因为"预测下一个字"这个目标的信息含量是无穷的。你永远可以预测得更准，而更准需要更深的理解。

## 三

兽卵要孵化，得有灵食。

灵食从哪来？从灵脉（互联网）上来。第一章讲过，半个世纪的灵脉铸造积累了天量的智元——网页、书籍、论坛帖子、代码仓库、维基百科、新闻、论文、社交媒体。

但生智元（原始数据）不能直接喂给神兽。就像你不能把一块矿石直接塞进嘴里——得先提炼。

**灵食炼制的流程是这样的：**

**第一步，采集。** 用爬虫从互联网上大规模抓取网页。Common Crawl 是修仙界最大的公共灵食矿——它每个月爬几十亿个网页，累计存储了几 PB 的文本。

**第二步，淬元（数据清洗）。** 抓回来的东西乱七八糟——广告、垃圾邮件、重复内容、乱码、色情、版权争议材料。得过滤。这一步不性感但极其重要。有研究表明，数据质量对最终效果的影响比数据数量更大。

**第三步，去重。** 同一段文字如果在训练数据里出现一百次，神兽就会对它"过度拟合"——死记经文。去重是防止走火入魔的基本功。

**第四步，切元术（Tokenizer）。** 这是最有意思的一步。

人类读文字是一个字一个字读的。但神兽不是——它读的是 Token（切元）。

Tokenizer 把文本切成一个个标准单元。怎么切？用 **BPE（Byte Pair Encoding）** 算法。

BPE 的思路很直觉：先把每个字符当成一个 Token。然后统计哪两个相邻 Token 出现得最频繁——比如"th"——就把它们合成一个新 Token。重复这个过程几万次，常见的词（"the""ing""tion"）变成了一个 Token，罕见的词被拆成多个 Token。

英文里 "understanding" 可能被切成 "under" + "standing" 两个 Token。中文里"人工智能"可能被切成"人工" + "智能"两个 Token，也可能是一个整 Token——取决于训练语料里这个词出现的频率。

不同门派的切元术不一样。GPT 用的 Tokenizer 跟 Llama 的不同，跟 DeepSeek 的也不同。切法不同意味着同一段文字切出来的 Token 数量不同——这直接影响训练效率和推理成本。

修仙界有句话：**"切元术定生死。"** 你的 Tokenizer 切得好，同样长的文本用更少的 Token 表示，训练更快、推理更便宜。切得差，别人一个 Token 搞定的事你要两个——效率白亏一半。

## 四

灵食准备好了，兽卵也有了。下一步是**封坛孵兽**。

一颗兽卵多大？GPT-3 有 1750 亿个凝智元（参数），用 BF16 精度存储就是 350 GB。一块 A100 灵核的灵池只有 80 GB。

你连兽卵都塞不进一颗灵核。

这就是为什么需要**合阵修炼**——把成百上千颗灵核组成一座灵坛，协同孵化。

合阵的方式有很多，详情见《合阵修炼六维并行》。这里只说大白话：

**数据并行（DP）**：复制多份兽卵，每份喂不同的灵食，算完同步心得。简单粗暴，但每颗灵核都要装一整颗兽卵。

**全分片并行（FSDP）**：把兽卵切碎，每颗灵核只存一小块。需要用的时候临时拼回来。省灵池，但通信压力大。

**张量并行（TP）**：把每一层的矩阵横着切，多颗灵核各算一半。灵核之间需要极快的经脉（NVLink）。

**流水线并行（PP）**：按层分段，前几层在这颗灵核，后几层在那颗。像工厂流水线一样。

实际的大规模封坛往往是四种并行混着用——所谓"四维混元阵"。DeepSeek V3 的 2048 颗 H800 灵坛就用了 DP + TP + PP + EP（专家并行）的组合。

封坛的规模感有多大？

DeepSeek V3：2048 颗 H800，封坛约两个月，总成本 557 万美元。
GPT-4：具体配置未公开，但据估算用了上万颗 A100/H100，训练数月，成本超过一亿美元。
Llama 3 405B：16384 颗 H100，训练消耗 3.93 × 10²⁵ FLOPS。

这些数字的背后是什么？是数据中心里成千上万块显卡日夜不停地运转，是几兆瓦的电力消耗，是液冷系统不间断地给灵核散热。封坛期间任何一颗灵核故障都可能导致训练中断——然后你得从上一个存档（Checkpoint）重新开始。

**孵化神兽，本质上是一场跟硬件故障赛跑的马拉松。**

## 五

封坛孵兽不是一帆风顺的。灵食灌注的过程中，随时可能出事。

**兽性反噬（Loss Spike）。** 训练进行到某个阶段，Loss 突然暴涨——就像兽卵里的幼兽突然暴走了。通常几百步之后会自己恢复。但严重的 spike 可能导致训练彻底崩溃。

为什么会 spike？原因很多——某一批数据质量特别差、学习率太大、某个梯度突然异常、数值精度溢出。就像孵化过程中突然喂了一口变质的灵食，幼兽消化不了，暴躁了。

**灵力暴走（梯度爆炸）。** 梯度在反向传播的过程中越来越大，最终变成 NaN（不是一个数字）。整个训练报废。

解法是**梯度裁剪（Gradient Clipping）**——给缰绳设一个上限，超过就截断。简单但有效。

**Learning Rate 的讲究。** 学习率太大，兽卵裂了。学习率太小，孵到天荒地老也孵不出来。

修仙界的标准做法是 **Warmup + Cosine Decay**——开始的时候慢慢加大灵力灌注（Warmup），到了峰值之后缓缓降低（Cosine Decay）。就像火候——起初慢慢升温让兽卵适应，中间猛火锻造，后期文火收功。

2025 年，Kimi 的 K2 团队提出了 **MuonClip**——一种让 Loss 全程零 spike 的训练技术。具体原理涉及优化器的动量校准，但效果是惊人的：整个训练过程平稳如镜，没有一次兽性反噬。

修仙界的评价：以前孵兽像坐过山车，现在 MuonClip 把过山车铺成了高速公路。

## 六

几个月的封坛之后，兽卵终于裂了。

一头新生的神兽破壳而出。

它会什么？

**它懂语言。** 你用任何语言跟它说话，它都能理解（前提是训练数据里有这种语言）。

**它有知识。** 训练数据里的事实、历史、科学、文化，它都吸收了。你问它"太阳的表面温度是多少"，它知道。

**它能推理。** 至少是初步的推理。"如果 A 大于 B，B 大于 C，那么 A 大于 C"——它能做到。

**它能生成。** 你给它一个开头，它能往下写出连贯的文章。

但它有一个致命的问题：**不听话。**

你让它写一首诗，它可能给你接一段维基百科。你问它"1+1=？"，它可能回答一个脑筋急转弯。你让它帮你改简历，它可能开始编故事。

因为预训练只教了它"语言是什么样的"，没教它"人类想要你怎么回答"。它是一头有灵智但没有灵契的野兽——聪明，但不受控。

修仙界把这个状态叫：**有灵智，无灵契。**

灵契怎么结？SFT 先教基本礼仪，RLHF/GRPO 再深度对齐。这就是后面五章的故事了。

预训练把一颗混沌兽卵变成了一头有灵智的幼兽。接下来的任务，是把这头幼兽变成一头**可用的**神兽。

**孵化只是开始。真正的修行，在破壳之后。**
