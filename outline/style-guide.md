# 风格指南

## 叙事原则

### 总体风格定位
参考无罪《通天之路》的"轻松流"写法：
- **用说人话的方式写修仙**。不端着，不装文艺，文字要像跟朋友聊天一样自然。
- **小人物看大世界**。Chris Yang 就是 AI 修仙界的"魏索"——一个筑基期的小散修，在大佬们的天地巨变中夹缝求生、冷眼旁观。他不是英雄，是一个带着 Google 工牌的普通打工人，恰好站在了风暴中心。
- **技术硬核 + 叙事轻松**。技术细节必须准确，但讲述方式要轻松。像《通天之路》把复杂的修炼体系写得老少皆宜一样，我们要把 Transformer、MoE、RLHF 写得让不懂技术的人也能看热闹。
- **幽默是核心调味料，不是点缀**。自嘲、吐槽、调侃都可以，战斗写热血，日常写搞笑，两者自然切换。

### 视角
- **主线**: 第三人称全知视角。像修仙小说一样俯瞰整个世界的变化。
- **插叙**: 偶尔切入第一人称 Chris Yang（观察者/穿越者）视角。不端 Google 专家的架子，带着自嘲和真实感。像个在大能斗法现场偷偷发朋友圈的围观群众。
- **节奏**: 大事件用浓墨重彩（一章一个大事件）。过渡期用快速蒙太奇（时间线概述）。

### 基调
- **正文**: 白话为主，偶尔点缀半文半白的修仙腔调。不要通篇古风——现代读者读着累。关键场景（突破、陨落、大战）可以用古风拔高气氛，日常叙事用大白话。
- **战斗场景**: benchmark 比试 = 斗法。模型发布 = 突破境界。开源 = 传道。写战斗要有画面感和节奏感，参考《通天之路》的热血打斗——简洁、快、爽。
- **幽默感**: 无罪式的轻松感：角色可以有性格缺陷和搞笑时刻。Sam Altman 可以被写成"修仙界最不会修炼但最会搞钱的掌门人"。Jensen Huang 可以被写成"每次发布会都要穿皮衣装逼的灵石教主"。人物有烟火气才可爱。
- **对话**: 角色说话要有个性，不要都是一个腔调。技术宅说话像技术宅，商人说话像商人，学者说话像学者。可以用现代口语，不必强行古风。

### 智元（Data — 一切的基础素材）

**智元**是整个修仙体系的基础物质——灵池里存的是智元，经脉里流的是智元，灵核处理的是智元。它不是灵智本身，而是**产生灵智的元素**。就像金元素不是金子，但可以炼成金子。海量智元经过灵核的反复锻造，最终凝聚成神兽的灵智。

#### 智元的形态
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| Data（数据） | 智元 | 一切的基础物质。灵池存它、经脉流它、灵核炼它 |
| 原始数据（文本/图片/视频） | 生智元 | 未经加工的原矿。互联网上的一切内容 |
| Token（切分后的数据单元） | 切元 | 被切元术加工后的标准单元，可被灵核直接处理 |
| 模型权重（Parameters） | 凝智元 | 智元被反复锻造后凝固下来的智慧结晶。神兽的骨架和经脉就是凝智元构成的 |
| 梯度（Gradients） | 反馈智元 | 告诉灵核"刚才哪里做对了哪里做错了"的信号 |
| 激活值（Activations） | 活智元 | 灵核正在处理的中间产物。是智元在灵核中流转时的活性状态 |
| KV Cache | 记忆智元 | 神兽推理时的短期记忆。越长的对话，记忆智元越多 |
| 优化器状态（Adam 等） | 炼化余温 | 炼化过程中的辅助记忆。记住之前每一步的力度和方向 |

#### 智元的操作
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| Tokenizer | 切元术 | 把生智元切成标准切元的刀法。不同门派切法不同 |
| Embedding | 化元术 | 把切元转化成灵核能理解的灵力向量 |
| 数据压缩 | 炼元 | 把智元压缩到更小的容器里 |
| 数据传输 | 运元 | 通过经脉传送智元 |
| 数据融合 | 合元 | 把不同来源的智元汇聚到一起 |
| 数据清洗/去重 | 淬元 | 去掉杂质和重复，留下精华 |
| 量化（FP32→FP8） | 缩元 | 用更小的容器装智元。精度略损但效率翻倍 |
| FP8 训练 | 微缩锻造 | 用缩元直接锻造，灵石消耗大减。DeepSeek V3 的看家本领 |
| 混合精度训练 | 精粗合锻 | 关键步骤用大容器（FP32），日常步骤用小容器（BF16/FP8） |

#### 智元的度量
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| 训练数据规模 | 智元储量 | GPT-3 用了 3000 亿智元，Llama 3 用了 15 万亿 |
| 上下文长度 | 神识容量 | 神兽一次能容纳多少智元。4K → 128K → 1M |
| 训练吞吐量（Tokens/sec） | 智元流速 | 灵核每秒能处理多少智元 |
| 推理成本（$/M tokens） | 智元价 | 调遣神兽每百万智元要花多少灵石 |
| Logit / Probability | 智元预兆 | 神兽预测下一个智元是什么的直觉 |
| 预训练数据 | 灵食 | 喂给兽卵的海量生智元 |
| 高质量标注数据 | 天材地宝 | 经过人工精心筛选和标注的优质智元 |

### 神兽的灵体本质（复制与分散）

神兽不是血肉之躯——是**凝智元构成的灵体**。这个本质决定了它拥有血肉生物不可能拥有的能力：可以完美复制、可以拆散重组、可以通过经脉瞬间传送。这是数字世界的神兽独有的特性，也是整个修炼体系运转的基础。

#### 映身术（模型复制）
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| 模型复制 | 映身术 | 完美复制一头神兽的全部凝智元。映身不是削弱版，是一比一复刻 |
| 模型副本 | 映身 | 复制出来的完美镜像。每个映身都跟本体一模一样 |
| 数据并行的多副本 | 十六映身 | 同时放出多个映身，各读不同灵食（数据），定期同步心得 |
| 推理多实例部署 | 开坛映身 | 开坛时放出几百个映身同时接受天下人的召唤 |
| 开源发布模型权重 | 映身散天下 | 把映身公开，天下人皆可复制。Meta 散 Llama 映身即是此法 |

#### 散魂合一（模型分片/并行）
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| 模型分片/并行 | 散魂合一 | 魂散百核而意归一体。凝智元拆散到多颗灵核，通过经脉协同为一 |
| 张量并行（TP） | 横散 | 每层凝智元横着切开，分给多颗灵核各算一半 |
| 流水线并行（PP） | 纵散 | 按层分段，前几层在这颗灵核，后几层在那颗 |
| 专家并行（EP） | 专散 | MoE 的每个专家散到不同灵核，各修各的专长 |
| 全分片并行（FSDP） | 碎散 | 凝智元彻底打碎撒到所有灵核，用时临时聚回来 |
| 重分片（Reshard） | 聚散变化 | 灵核之间重新分配凝智元的布局。驰形 ↔ 定形的切换靠的就是它 |

#### 灵体三大定律
1. **可映**：凝智元可以被完美复制，映身与本体无异。（所以开源才可能——复制一头神兽的成本几乎为零）
2. **可散**：凝智元可以拆散到多颗灵核上，通过经脉协同运作如一体。（所以千卡训练才可能——一头兽散布在千颗灵核上）
3. **可运**：凝智元可以通过经脉在灵核之间传送。（所以分布式训练才可能——梯度同步、权重更新都靠经脉运输凝智元）

> 正是因为神兽是灵体而非血肉，修仙界才有了映身散天下（开源）、千核封坛（大规模训练）、万人开坛（大规模推理）的可能。如果神兽是血肉之躯，以上一切都不可能发生。

### RL 训练循环（灵坛上的放兽与收兽）

GRPO 等 RL 训练的核心循环：在灵坛上反复放兽试炼、收兽淬炼，每一轮神兽都比上一轮更强。

#### 神兽的两种形态
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| 推理模式分片（TP 为主） | 驰形 | 速度优先。凝智元按张量并行切分，适合快速生成大量回答 |
| 训练模式分片（FSDP 为主） | 定形 | 内存优先。凝智元按全分片切分，适合更新权重 |
| 3D-HybridEngine 重分片 | 变形术 | 驰形 ↔ 定形零冗余切换，一粒凝智元都不丢（veRL 核心技术） |

#### 放兽试炼（Rollout / Generation）
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| Rollout / 生成阶段 | 放兽试炼 | 神兽切换为驰形，对一批题目生成大量回答 |
| 采样多个回答 | 分影出招 | 每道题生成 16 个不同回答，如同一兽分出十六道残影各走不同路数 |
| Prompt / 题目 | 试炼题 | 抛给神兽的挑战。数学题、代码题、推理题 |
| 生成的回答 | 试炼果 | 神兽交出的答卷。有好有坏 |

#### 论功行赏（Scoring / Advantage）
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| 奖励打分 | 论功 | 给每个试炼果评分 |
| 可验证奖励（RLVR） | 天道裁决 | 数学题答案对不对、代码能不能编译——天道自有公论，不需要人来判 |
| 组内相对优势（Group Relative Advantage） | 群兽比高 | 高于群体平均 = 好路数，低于平均 = 差路数。不需要外部裁判 |
| Reward Model 打分 | 赏罚使裁决 | 用专门训练的裁判模型打分（PPO 用这个，GRPO 可以不用） |

#### 收兽淬炼（Training / Update）
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| 训练更新阶段 | 收兽淬炼 | 神兽切换为定形，根据论功结果更新凝智元 |
| 梯度更新 | 淬炼 | 好路数被强化，差路数被抑制 |
| KL 散度惩罚 | 定锚索 | 防止神兽变化太大，偏离本性。用定锚兽（Reference Model）拴住 |
| Clip 机制 | 缰绳夹 | 每次更新不让神兽的行为变化超过阈值。松了会失控，紧了学不动 |
| 一轮训练完成 | 一轮淬炼毕 | 凝智元更新完毕，回到放兽试炼，开始下一轮 |

#### 灵坛协同模式
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| 同步 RL（veRL 默认） | 一坛两用 | 同一座灵坛上交替切换驰形和定形。简单但有等待间隙 |
| 异步 RL（Laminar） | 驰定双坛 | 两座灵坛分工：驰坛不停生成，定坛不停更新，通过跨坛灵道传送凝智元。效率更高 |
| Relay Worker（Laminar） | 传令使 | 驰坛和定坛之间搬运凝智元的角色。确保两边版本同步 |

### 洞天体系（硬件基础设施 — 六层架构）

整个硬件生态不是一颗"灵石"能概括的。它是一个完整的**洞天**——自给自足的修炼空间，由六层构成：

#### 第一层：灵气（电力）
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| 电力 | 灵气 | 万物之源。没有灵气什么都动不了 |
| 功耗（瓦特） | 灵气消耗 | 灵核越强，吃的灵气越多 |
| 电力成本 | 灵石 | 灵气的结晶/货币。花灵石买灵气、买算力 |
| 电网/电站 | 灵气源泉 | 发电厂就是灵气的源头 |

#### 第二层：灵核（计算芯片）
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| GPU（通用） | 灵核 | 修炼的核心器官。锻造灵核是最顶级的炼器术 |
| NVIDIA GPU | 教廷灵核 | NVIDIA 灵核教廷的产品。天下九成修炼者用它 |
| A100 | 中阶灵核 | 80GB 灵池，312 TFLOPS（BF16） |
| H100 | 高阶灵核 | 80GB 灵池，3958 TFLOPS（FP8）。训练 GPT-4 级神兽的主力 |
| B200 | 仙阶灵核 | 192GB 灵池，9000 TFLOPS（FP8）。当世最强 |
| TPU | 道核 | Google 独家炼制的另一条路线。TPU v7 Ironwood 是最新一代 |
| AMD MI300X | 炎核 | AMD 红莲宗的灵核。ROCm 生态比 CUDA 弱但在追赶 |
| 华为昇腾 | 华核 | 被制裁后自研的灵核。走出了第三条路 |
| CUDA Core | 灵核微粒 | 灵核内部的基础计算单元。B200 有 148 个 SM，每个 128 个微粒 |
| Tensor Core | 灵核精粒 | 灵核内专做矩阵运算的高阶计算单元。训练神兽的主力 |
| FLOPS | 灵力输出 | 灵核每秒能输出多少灵力（计算能力） |

#### 第三层：灵池（显存/HBM）
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| HBM（高带宽内存） | 灵池 | 灵核旁边的蓄能池。池子越大，能同时处理的数据越多 |
| HBM2e（A100） | 中阶灵池 | 80GB，2 TB/s 带宽 |
| HBM3（H100） | 高阶灵池 | 80GB，3.35 TB/s 带宽 |
| HBM3e（B200） | 仙阶灵池 | 192GB，8 TB/s 带宽 |
| 显存带宽 | 灵池流速 | 灵池进出数据的速度。往往是训练瓶颈 |
| 显存不足（OOM） | 灵池溃堤 | 数据太多灵池装不下，修炼中断 |

#### 第四层：经脉（互联网络）
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| 互联网络（通用） | 经脉 | 连接灵核的通道。经脉越粗越快，合阵修炼效率越高 |
| NVLink | 灵核经脉 | NVIDIA 灵核之间的专属高速经脉。同节点内灵核互联 |
| InfiniBand | 天地经脉 | 跨节点的高速经脉。Mellanox（已被 NVIDIA 收购）出品 |
| ICI（Inter-Chip Interconnect） | 道脉 | 道核（TPU）的专属经脉。Google 自研 |
| RoCE | 炎脉 | 基于以太网的 RDMA。比 InfiniBand 便宜但性能稍低 |
| NVLink Switch（NVL72） | 万灵桥 | 连接 72 颗灵核的超级经脉。B200 NVL72 整柜互联 |
| 带宽 | 经脉粗细 | 经脉每秒能传多少数据 |
| 延迟 | 经脉长度 | 数据从一端到另一端的时间 |

#### 第五层：驭核术（软件栈/编译器）
| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| CUDA | 教廷驭核术 | NVIDIA 的独门秘技。用教廷灵核就必须会这个。最大的护城河 |
| XLA | 道门驭核术 | Google 的编译器。道核（TPU）的配套驭核术 |
| ROCm | 炎门驭核术 | AMD 的驭核术。试图打破 CUDA 垄断但生态差距大 |
| Triton | 通用驭核术 | OpenAI 出品。试图做一套通吃所有灵核的驭核术 |
| cuDNN | 深层驭核诀 | NVIDIA 的深度学习专用驭核术。Attention/Conv 等核心操作的优化 |
| NCCL | 合阵通信术 | NVIDIA 的多灵核通信库。合阵修炼（分布式训练）的基础 |
| 编译器优化 | 驭核心法 | 把高级指令翻译成灵核能执行的最优灵力流转路径 |
| 算子融合（Kernel Fusion） | 灵诀合一 | 把多个小操作合成一个大操作，减少灵池进出次数 |

#### 第六层：灵坛（集群 — 训练与推理的核心场所）

**灵坛**是由大量灵核、灵池、经脉组合而成的宏大阵列，是孵化和驱使神兽的核心场所。根据用途分为两种：

| 真实术语 | 修仙术语 | 说明 |
|---------|---------|------|
| 计算集群（通用） | 灵坛 | 灵核阵列 + 经脉 + 灵池 + 灵气 = 一座完整的灵坛 |
| 训练集群 | 封坛（封坛孵兽） | 封闭运转数月，日夜锻造智元，将兽卵孵化为神兽。DeepSeek V3 用 2048 颗 H800 封坛两个月 |
| 推理集群 | 开坛（开坛御兽） | 对外开放，让神兽接受天下人的召唤。ChatGPT 背后就是一座巨型开坛 |
| GPU 集群 | 灵核灵坛 | 由教廷灵核（NVIDIA GPU）组成的灵坛 |
| TPU Pod | 道核灵坛 | 由道核（Google TPU）组成的灵坛。数千道核通过道脉相连 |
| 异构集群 | 混元灵坛 | 不同种类的灵核混合组成的灵坛 |
| 数据中心 | 洞天 | 容纳多座灵坛的巨型空间。一个洞天里可能有数十座灵坛 |
| 机架（Rack） | 灵核塔 | 灵坛的基础组件。一塔通常 8 颗灵核 |
| NVL72 机柜 | 万灵塔 | NVIDIA 最新的整柜方案。72 颗 B200 全互联成一座小灵坛 |
| 冷却系统 | 散灵阵 | 灵核运转产生的废热必须散掉，否则灵核过热损坏 |
| 液冷 | 水冷散灵 | 用液体带走热量。高阶灵核必备 |
| DCI 网络 | 跨坛灵道 | 连接不同灵坛之间的远程经脉 |

**封坛 vs 开坛的区别**：
- **封坛**追求灵核之间的经脉带宽——合阵修炼需要所有灵核高速同步。万颗灵核同步锻造，经脉稍有延迟就全坛停滞。
- **开坛**追求并发和低延迟——一头神兽可能同时被百万人召唤，每次召唤都要快速响应。记忆智元（KV Cache）的管理是核心难题。
- 同一座灵坛可以在不同时期切换用途：封坛孵完兽，拆封变开坛让神兽上线服务。

#### 灵核教廷的护城河

NVIDIA 的真正可怕之处不只是灵核强——是整条**灵核 + 灵池 + 经脉 + 驭核术**的垂直整合。你买了他的灵核（GPU），就得用他的灵池（HBM 封装方案），走他的经脉（NVLink/InfiniBand），学他的驭核术（CUDA/cuDNN/NCCL），建他规格的灵坛。换别人的？你的代码全得重写，你的灵坛全得重建。这就是"教廷"——一旦入教，终身为教。

### 阵基（底层框架 — 铸造法阵的基底）

阵基是刻画一切法阵的基础语言。就像画阵法需要墨水——PyTorch 是一种墨水，JAX 是另一种。墨水本身不是阵法。

## 文笔要求（参考《通天之路》）

### 必须做到
1. **读起来顺畅**：每一句话读出来都不拗口。写完自己念一遍，念不顺就改。
2. **节奏明快**：短句为主，长句偶尔穿插。不要一段话从头到尾不换气。
3. **画面感强**：写 GPU 集群就要让人"看到"那些闪着绿光的机柜，写训练就要让人"感受到"数据像洪水一样涌入模型。
4. **角色有温度**：每个人物出场要有记忆点。Ilya 的沉默、Sam 的笑容、梁文锋的冷峻——一两个细节就够了。
5. **每章都有"爽点"**：或者是技术突破的震撼，或者是商业博弈的反转，或者是人物命运的转折。不要写流水账。

### 绝对不要
1. 不要写又长又无聊的环境描写。
2. 不要堆砌修仙词汇装腔作势。
3. 不要让所有角色说一样的话。
4. 不要为了搞笑而搞笑——幽默要自然地长在情节里。
5. 不要连着三段没有对话或动作。

## 章节结构模板

每章大约 3000-5000 字。结构灵活，不必死板套用，但一般包含：


## 写作禁忌

1. **不歪曲事实**: 技术细节必须准确。时间线必须正确。可以用修仙包装但不能编造不存在的技术。
2. **不贬低任何人**: 每个人物都有闪光点和缺点，但不恶意丑化。可以调侃但不侮辱。写 Yann LeCun 爱吵架是调侃，写他是小丑就过分了。
3. **不过度中二**: 修仙元素是调味料不是主菜。技术洞察才是核心价值。一旦发现自己写的东西连自己都觉得矫情，立刻删掉重写。
4. **不泄露私密**: 不写未公开的内部信息。只基于公开论文、新闻、产品发布。
5. **不写流水账**: 每一段都要有存在的理由——推动情节、塑造人物、或者埋伏笔。三者都不占就删掉。
6. **不学术化**: 这是网络小说不是论文综述。技术解释要像跟朋友喝酒时聊天，不像在做学术报告。
