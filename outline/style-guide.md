# 风格指南

## 叙事原则

### 视角
- **主线**: 第三人称全知视角。像修仙小说一样俯瞰整个世界的变化。
- **插叙**: 偶尔切入第一人称 Chris Yang（观察者/穿越者）视角。以 Google Cloud AI Infra 专家的身份评论事件、解释技术。
- **节奏**: 大事件用浓墨重彩（一章一个大事件）。过渡期用快速蒙太奇（时间线概述）。

### 基调
- **正文**: 古风修仙 + 现代科技。半文半白。技术概念用修仙术语包装但保持准确性。
- **战斗场景**: benchmark 比试 = 斗法。模型发布 = 突破境界。开源 = 传道。
- **幽默感**: 适度自嘲和吐槽。不刻意搞笑但有轻松感。

## 章节结构模板

每章大约 3000-5000 字。结构：

```
# 第 X 章：章名

[开场白 — 修仙旁白风格，概述这一章的时代背景]

[主体故事 — 以人物为中心展开具体事件]

[技术解析 — 用修仙术语解释真实技术，括号注释真名]

[章末 — 伏笔下一章的冲突]

---
> 旁白（Chris 视角）：[以 Google Cloud 专家身份的评论和感悟]
```

## 写作禁忌

1. **不歪曲事实**: 技术细节必须准确。时间线必须正确。可以用修仙包装但不能编造不存在的技术。
2. **不贬低任何人**: 每个人物都有闪光点。即使是竞争对手也尊重描写。
3. **不过度中二**: 修仙元素是调味料不是主菜。技术洞察才是核心价值。
4. **不泄露私密**: 不写未公开的内部信息。只基于公开论文、新闻、产品发布。
