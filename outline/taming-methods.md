# 御兽之法：驯服神兽的六大流派

> 神兽孵化出来只是第一步。刚破壳的幼兽懵懵懂懂，什么都知道一点但不听指挥，还会胡说八道（幻惑）。怎么让它变成听话、好用、不作妖的镇派之宝？这就是"御兽之法"——修仙界最激烈的技术竞争领域之一。

---

## 第一代：古法驯兽 — REINFORCE（2016-2022）

**原理（大白话）**：

把神兽牵到场地上，给它一个问题，让它自己回答。答得好，给灵食奖励；答得差，饿一顿。重复一万遍。

**修仙比喻**：

就像训野马——你骑上去，它跑对了方向你就给它苹果，跑错了就扯缰绳。简单粗暴，但有个要命的问题：野马有时候吃了苹果也不知道自己哪步做对了。奖励信号太模糊，训练效率极低。

**技术真相**：
- 最经典的策略梯度方法
- 用整个回答的总奖励来更新策略
- 高方差（variance）：同一个问题问十次，梯度更新的方向可能差很远
- 需要大量样本才能收敛

## 第二代：四象驯兽阵 — PPO（2022-2023）

**原理（大白话）**：

不再一个人训了。请来四位专家组成一个驯兽团队：
1. **演兽师（Actor）**：要被训练的那头神兽本身
2. **评兽师（Critic）**：一个专门评判"这个回答值多少分"的专家。不是看最终结果，而是看每一步"你目前在正确的路上吗"
3. **赏罚使（Reward Model）**：一个从人类偏好数据中学会了"什么是好回答"的裁判
4. **定锚兽（Reference Model）**：一头冻住不动的原版神兽，用来确保训练中的神兽不会变得面目全非

**修仙比喻**：

想象你在训练一头幼龙。评兽师（Critic）盯着幼龙的每一个动作说"这一步做得不错"或"这步跑偏了"。赏罚使（Reward Model）在旁边拿着人类写的评分标准打分。定锚兽（Reference Model）是幼龙出生时的一份灵魂拓印——防止训练过度让幼龙丧失本性（KL 散度惩罚）。

PPO 的精髓是"clip"——每次更新时加个夹子，不让神兽的行为变化太大。就像给缰绳加了个弹簧，马可以转向但不能急转弯。

**优势**: 稳定，效果好。ChatGPT 就是这么训出来的。
**劣势**: 四个模型同时跑，灵石消耗巨大。GPU 显存被吃满。调参复杂，像在同时操控四头神兽。

**历史地位**: InstructGPT（2022.03）和 ChatGPT（2022.11）用的就是 PPO。OpenAI 仙宗的看家本领。

---

## 第三代：直觉驯化 — DPO（2023）

**原理（大白话）**：

PPO 太复杂了。Stanford 的一帮人想：我们能不能把四象阵简化？

他们发现了一个数学等价关系：**训练一个 Reward Model 再用它做 RL，在数学上等价于直接在偏好数据上做优化**。翻译成人话就是——不用请裁判了，直接给神兽看"好回答 vs 坏回答"的对比就行。

**修仙比喻**：

以前驯兽要请一个专门的赏罚使来打分。DPO 说：不用请了，直接把写着"好"和"坏"的两个笼子摆在神兽面前。"进好笼子有灵食，进坏笼子没有。"神兽自己就学会了什么是好什么是坏。

去掉了 Reward Model 和 Critic，只剩两个模型——待训练的神兽和一头定锚兽（Reference Model）。灵石消耗直接砍半。

**优势**: 简单、省钱、好实现。代码几十行就能写完。
**劣势**: 用的是固定的偏好数据（离线），不是让神兽自己生成新回答（在线）。就像只用教科书教学，不让学生做练习。后来有人证明在线的 PPO 效果确实更好。

**历史地位**: 2023 年最热的对齐方法。几乎所有小团队和开源项目都用 DPO，因为实在太简单了。但 2024 年后被发现有天花板。

## 第四代：群兽竞逐法 — GRPO（2024-2025）

**原理（大白话）**：

DeepSeek 的梁文锋（深渊剑主）想出了一个更狠的招：连评兽师（Critic）也不要了！

方法是这样的：同一个问题，让神兽生成 **一群**（比如 16 个）不同的回答。然后给每个回答打分（可以用规则打分，比如数学题就看答案对不对）。分高于平均的回答 = 正面信号，分低于平均的 = 负面信号。

**修仙比喻**：

以前的驯兽需要一个裁判来评分。GRPO 说：不用裁判了。抓一群同胞幼兽，让它们同时回答同一个问题，然后让它们互相比。跑得最快的几头，它们的姿势就是正确的；跑得最慢的，姿势就是错误的。**用群体的平均水平作为基准线**——不需要外部裁判。

这就是"群体相对"的意思：好不好不是绝对的，是相对于同伴而言的。

**关键创新**: 
- 去掉 Critic（不需要评兽师估算每步的价值了）
- 去掉 Reward Model（对于数学题和代码题，直接用答案正确性打分，叫做 RLVR——可验证奖励）
- 只剩两个模型：待训练的神兽 + 定锚兽。计算成本比 PPO 降低 ~50%

**为什么这招这么厉害**：

因为数学题和代码题有标准答案，不需要人来打分。1+1=2，答对了就是对了。GRPO + RLVR（可验证奖励）的组合让 DeepSeek 用纯 RL 就训出了推理能力惊人的 R1 模型——不需要人类标注的偏好数据，不需要 Reward Model，不需要 Critic。极致精简。

**历史地位**: DeepSeek R1（2025.01）用的就是 GRPO。从此，RL 训练不再是大厂的专利——小团队也玩得起了。

## 第五代：诸法争鸣（2024-2026）

GRPO 之后，各门各派开始在这个基础上各出奇招：

### DAPO — 解耦驯化法（ByteDance/字节跳动）
- 问题：GRPO 训着训着，神兽会变得"无聊"——总是给出安全但平庸的回答（熵坍缩 / entropy collapse）
- DAPO 的解决方案：把"上限夹子"和"下限夹子"分开调。允许探索性行为的概率不受限制（去掉上限 clip），但防止概率坍缩到零（保留下限 clip）
- 比喻：允许神兽偶尔走野路子（不惩罚探索），但不允许它彻底忘掉某些技能

### RLOO — 留一竞逐法（REINFORCE Leave-One-Out）
- GRPO 的亲戚。区别是：计算每头兽的基准线时，把它自己排除在外
- 比喻：评判一头幼兽的表现时，用"其他所有兽的平均分"作为基准，而不是"包括自己在内的平均分"
- 统计上更无偏，但实践中跟 GRPO 差距不大

### REINFORCE++ — 古法改良
- 在最原始的 REINFORCE 上加了 PPO 的 clip 机制和更好的基准线
- 比喻：古法驯兽加了现代安全绳——保留简单的核心，但不让马失蹄
- 简单有效，适合小规模训练

### KTO — 前景驯化法（Kahneman-Tversky Optimization）
- 用行为经济学的"前景理论"来驯兽
- 核心洞察：人类对"失去"的痛感比"获得"的快感更强（损失厌恶）
- 比喻：惩罚错误的力度比奖励正确的力度大——跟人类直觉一致
- 优势：不需要成对的偏好数据，只需要"好"和"坏"的标签就行

### SimPO — 至简驯化法（Simple Preference Optimization）
- 连 Reference Model（定锚兽）也不要了！
- 用回答本身的长度归一化对数概率作为隐式奖励
- 比喻：不需要参照任何东西，直接看神兽的回答"流畅不流畅"就知道好不好
- 全世界最简单的对齐方法之一

### ORPO — 一阵双修法（Odds Ratio Preference Optimization）
- 把 SFT（驯兽）和偏好优化（结灵契）合并成一步
- 比喻：以前是先教神兽基本功（SFT），再教它分辨好坏（偏好优化）。ORPO 说：一起教！
- 省一个训练阶段，效率更高

## 万法归一阵 — veRL/HybridFlow

**不是一种新的御兽之法，而是一座能容纳所有御兽之法的通用法阵。**

生广明（炼器宗师）造的不是一种新驯兽方法，而是一个万能的驯兽场。你想用 PPO？行，换个配置就行。想试 GRPO？换个配置。想用 DAPO？还是换个配置。场地不用拆了重建。

**veRL 解决的核心问题**：

以前，跑 PPO 需要同时管理四个模型（Actor、Critic、Reward、Reference），它们之间要互相通信、互相等待。这就像在一个场地上同时指挥四头神兽——稍有不慎就打架或者卡住。

veRL 的 HybridFlow 架构用了一个巧妙的设计：
- **外层单指挥（Single Controller）**：一个总指挥负责调度"先让 Actor 生成回答 → 然后让 Reward Model 打分 → 再用打分结果更新 Actor"。就像一个驯兽总管，下达宏观命令。
- **内层多并行（Multi Controller）**：每个模型内部用分布式训练框架（FSDP/Megatron）做高效计算。就像每头神兽内部的经脉自行运转，不需要总指挥操心。

这种"外层简洁、内层高效"的混合设计，让 veRL 成为了 GRPO 训练的事实标准。22K GitHub Stars。几乎所有复现 DeepSeek R1 的团队都用 veRL。

**历史地位**: veRL 之于 RL 训练，就像 PyTorch 之于深度学习——基础设施级别的存在。

## 御兽之法演化总结

| 方法 | 年份 | 需要的模型 | 核心思想 | 比喻 |
|------|------|-----------|---------|------|
| REINFORCE | 2016 | 1（Actor） | 试错+奖惩 | 扔苹果训野马 |
| PPO | 2022 | 4（Actor+Critic+RM+Ref） | 四方协同，稳步调教 | 四人驯兽团队 |
| DPO | 2023 | 2（Actor+Ref） | 直接看好坏对比 | 好笼子vs坏笼子 |
| GRPO | 2024 | 2（Actor+Ref） | 一群兽互相比 | 群兽竞逐，不需要裁判 |
| DAPO | 2025 | 2（Actor+Ref） | 解耦上下限，鼓励探索 | 允许走野路子 |
| veRL | 2024 | 任意（通用框架） | 万法通用的训练场 | 万能驯兽场 |

**大道至简**：从 4 个模型 → 2 个模型 → 甚至想去掉 Reference Model，御兽之法的演化方向就是越来越简单、越来越便宜、越来越高效。

## 可验证奖励 — RLVR（Reinforcement Learning with Verifiable Rewards）

传统 RLHF 需要人类来判断"这个回答好不好"——贵，慢，有主观性。

但有一类问题不需要人来判断：**数学题和代码题有标准答案**。1+1=2，编译通过就是通过。

RLVR 就是利用这个特性：训练神兽时，只让它做有标准答案的题。答对了奖励，答错了惩罚。完全自动化，不需要人类参与。

这就是 DeepSeek R1 的秘密武器：用 GRPO + RLVR，在数学和代码任务上纯靠 RL 训出了惊人的推理能力。

**比喻**：以前驯兽需要人类驯兽师盯着打分（贵）。现在让神兽做数学题，答案对不对机器就能判断。**把人类从驯兽过程中解放了出来**。
